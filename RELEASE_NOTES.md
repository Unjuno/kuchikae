# Kuchikae v0.1.0 Release Notes

**Release date:** 2026-06-13

## Highlights

Kuchikae v0.1 is a prompt-conditioned, voice-conditioned speech transformation prototype for Japanese.

> Speak once. Kuchikae says it back in your voice, following your prompt.

## What's new

### CLI interface

- `kuchikae serve` — start the web server
- `kuchikae doctor` — check backend availability
- `kuchikae serve --dummy` — smoke test with dummy backends
- `kuchikae serve --real` — use real STT/TTS backends
- `kuchikae serve --streaming` — enable streaming STT for push-to-talk

### Voice style detection

- **Text-based rules**: detects urgent, apologetic, warm, or serious tone from transcript
- **Audio emotion**: parallel analysis of speaker's voice (happy, sad, calm, neutral)
- **Fusion**: combines text style, audio emotion, and LLM detection with confidence weighting
- **VoiceOutputPrompt** generated automatically from detected style

### Pipeline architecture

- `VoicePromptResolver` handles voice style detection and fusion
- `_step_text()` method manages cache, validation, and fallback
- Parallel audio emotion detection (ThreadPoolExecutor)
- Streaming STT for real-time push-to-talk

### Backends

- **STT**: faster-whisper (real), streaming, segmented
- **Text transform**: rule-based (fast), Ollama LLM (flexible), prompted rule (custom)
- **Voice output**: Irodori-TTS (real), OpenVoice (beta), dummy (smoke test)

### Testing

- 305 tests (291 passing, 14 skipped)
- Contract tests for backend conformance
- Golden tests for text quality
- E2E smoke tests
- Unit tests for all domain logic

## Quick start

### Smoke test (no models required)

```bash
uv run python -m kuchikae.cli serve --dummy
```

Opens Gradio at `http://127.0.0.1:7860`. Record audio → get dummy output.

### Real backends

```bash
uv sync --extra real
uv run python -m kuchikae.cli serve --real
```

Requires:
- faster-whisper models (auto-downloaded)
- Ollama running with a model
- Irodori-TTS models

### Check backends

```bash
uv run python -m kuchikae.cli doctor
```

Shows installed backends, Ollama status, and missing dependencies.

## Environment variables

| Variable | Description | Default |
|----------|-------------|---------|
| `KUCHIKAE_STT_BACKEND` | STT backend | `faster_whisper` |
| `KUCHIKAE_TEXT_BACKEND` | Text transform backend | `prompted_rule` |
| `KUCHIKAE_VOICE_BACKEND` | Voice output backend | `irodori` |
| `KUCHIKAE_STREAMING_STT` | Enable streaming STT | `0` |
| `KUCHIKAE_TEXT_MODEL` | Ollama model name | `qwen3:8b` |
| `KUCHIKAE_ALLOW_DUMMY_BACKENDS` | Allow dummy in real mode | `0` |

## Known limitations

- Voice prompt must not be freely generated by large LLMs (guardrails enforced)
- Audio emotion detection requires GPU for reasonable speed
- OpenVoice backend is beta quality
